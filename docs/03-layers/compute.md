# Compute Layer

The compute layer defines virtual machines and instance types.

## File Structure

```
layers/compute/
├── core-compute.yaml    # Auto-generated infrastructure VMs
└── compute.yaml         # Your custom VMs
```

Configure in `platform.yaml`:
```yaml
layers:
  compute: "layers/compute/core-compute.yaml,layers/compute/compute.yaml"
```

## Schema

Defined by [`ComputeConfig`](../08-reference/types/ComputeConfig.md) interface.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `instance_type_definitions` | [`ComputeType[]`](#computetype) | ✅ | Reusable instance types |
| `virtual_machines` | [`(VMBasedOnType \| VMCustom)[]`](#virtual-machines) | ✅ | VM definitions |
| `linux_containers` | [`VMCustom[]`](#vmcustom) | ❌ | LXC containers |

## ComputeType

Reusable instance type definition.

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `name` | `string` | ✅ | Unique type name |
| `cpu` | `number` | ✅ | Number of vCPUs |
| `ram` | `number` | ✅ | RAM in GB |
| `disk` | `number` | ✅ | Disk size in GB |
| `disk_type` | `"distributed"` \| `"local"` | ✅ | Storage type |
| `os_template` | `string` | ✅ | OS template name or URL |
| `is_gpu_enabled` | `boolean` | ❌ | Enable GPU passthrough |

### Example

```yaml
instance_type_definitions:
  - name: database-prod
    cpu: 4
    ram: 8
    disk: 150
    disk_type: distributed
    os_template: debian-12-cloudinit

  - name: app-medium
    cpu: 4
    ram: 8
    disk: 100
    disk_type: distributed
    os_template: debian-12-cloudinit

  - name: worker-gpu
    cpu: 8
    ram: 32
    disk: 200
    disk_type: local
    os_template: ubuntu-24.04-cloudinit
    is_gpu_enabled: true
```

## Virtual Machines

Two types: `VMBasedOnType` (recommended) and `VMCustom` (local only).

### VMBasedOnType

References an instance type definition. **Required for production/enterprise.**

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `name` | `string` | ✅ | Unique VM name |
| `vm_id` | `number` | ✅ | Proxmox VM ID |
| `host` | `string` | ✅ | Target Proxmox node |
| `role` | [`VMRole`](../08-reference/types/VMRole.md) | ✅ | VM role |
| `type_definition` | `string` | ✅ | Reference to ComputeType.name |
| `public_ip` | `string` | ❌ | Public IP assignment |
| `status` | `"running"` \| `"stopped"` \| `"provisioning"` | ❌ | Current status |

### VMCustom

Inline specs. **Only allowed in local tier.**

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `name` | `string` | ✅ | Unique VM name |
| `vm_id` | `number` | ✅ | Proxmox VM ID |
| `host` | `string` | ✅ | Target Proxmox node |
| `role` | [`VMRole`](../08-reference/types/VMRole.md) | ✅ | VM role |
| `cpu` | `number` | ✅ | Number of vCPUs |
| `ram` | `number` | ✅ | RAM in GB |
| `disk` | `number` | ✅ | Disk size in GB |
| `disk_type` | `"distributed"` \| `"local"` | ✅ | Storage type |
| `os_template` | `string` | ✅ | OS template |

## Complete Example

### core-compute.yaml (Auto-generated)

```yaml
# DO NOT MODIFY - Auto-generated by Soverstack

instance_type_definitions:
  - name: firewall-prod
    cpu: 4
    ram: 4
    disk: 20
    disk_type: local
    os_template: vyos-1.4

  - name: database-prod
    cpu: 4
    ram: 8
    disk: 150
    disk_type: distributed
    os_template: debian-12-cloudinit

virtual_machines:
  - name: vyos-01
    vm_id: 1
    host: pve-01
    role: firewall
    type_definition: firewall-prod

  - name: vyos-02
    vm_id: 2
    host: pve-02
    role: firewall
    type_definition: firewall-prod

  - name: postgres-01
    vm_id: 250
    host: pve-01
    role: database
    type_definition: database-prod
  # ... more VMs
```

### compute.yaml (Your custom VMs)

```yaml
# Your custom VMs

instance_type_definitions:
  - name: app-small
    cpu: 2
    ram: 4
    disk: 50
    disk_type: distributed
    os_template: debian-12-cloudinit

  - name: app-large
    cpu: 8
    ram: 16
    disk: 200
    disk_type: distributed
    os_template: debian-12-cloudinit

virtual_machines:
  - name: myapp-web-01
    vm_id: 3001
    host: pve-01
    role: general_purpose
    type_definition: app-small

  - name: myapp-web-02
    vm_id: 3002
    host: pve-02
    role: general_purpose
    type_definition: app-small

  - name: myapp-api-01
    vm_id: 3010
    host: pve-03
    role: general_purpose
    type_definition: app-large

linux_containers: []
```

## Validation Rules

### All Tiers

| Rule | Severity |
|------|----------|
| Unique VM names | Critical |
| Unique VM IDs | Critical |
| VM ID in correct range for role | Error |
| Referenced type_definition exists | Error |
| CPU > 0 | Error |
| RAM > 0 | Error |
| Disk > 0 | Error |

### Production/Enterprise

| Rule | Severity |
|------|----------|
| VMCustom not allowed | Critical |
| Mandatory VMs present | Critical |
| Minimum VM count per role | Critical |
| Resource specs meet requirements | Warning |

## Merge Behavior

When multiple files specified:

```yaml
compute: "core-compute.yaml,compute.yaml"
```

| Property | Behavior |
|----------|----------|
| `instance_type_definitions` | Concatenated |
| `virtual_machines` | Concatenated |
| `linux_containers` | Concatenated |

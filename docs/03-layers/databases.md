# Databases Layer

The databases layer defines PostgreSQL clusters and their databases.

## File Structure

```
layers/databases/
├── core-databases.yaml    # Mandatory databases (auto-generated)
└── databases.yaml         # Your custom databases
```

Configure in `platform.yaml`:
```yaml
layers:
  database: "layers/databases/core-databases.yaml,layers/databases/databases.yaml"
```

## Schema

Defined by [`DatabasesLayer`](../08-reference/types/DatabasesLayer.md) interface.

## Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `databases` | [`DatabaseCluster[]`](#databasecluster) | ✅ | Array of database clusters |

## DatabaseCluster

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `type` | `"postgresql"` | ✅ | Database type (only PostgreSQL) |
| `version` | `"14"` \| `"15"` \| `"16"` | ✅ | PostgreSQL version |
| `cluster` | [`ClusterConfig`](#cluster-config) | ✅ | Cluster configuration |
| `port` | `number` | ❌ | Port (default: 5432) |
| `ssl` | `"required"` \| `"preferred"` \| `"disabled"` | ✅ | SSL mode |
| `databases` | [`DatabaseDefinition[]`](#database-definition) | ✅ | Databases to create |
| `credentials` | [`CredentialRef`](../08-reference/types/CredentialRef.md) | ✅ | Admin credentials |
| `backup` | [`BackupConfig`](#backup-config) | ❌ | Backup configuration |

### Cluster Config

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `name` | `string` | ✅ | Unique cluster name |
| `ha` | `boolean` | ✅ | Enable HA (Patroni) |
| `vm_ids` | `number[]` | ✅ | PostgreSQL VM IDs |
| `read_replicas_vm_ids` | `number[]` | ❌ | Read replica VM IDs |

### Database Definition

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `name` | `string` | ✅ | Database name |
| `owner` | `string` | ✅ | Database owner/user |

### Backup Config

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `storage_backend` | `string` | ✅ | Reference to storage backend |
| `schedule` | `string` | ✅ | Cron expression |
| `retention.daily` | `number` | ✅ | Daily backups to keep |
| `retention.weekly` | `number` | ✅ | Weekly backups to keep |
| `retention.monthly` | `number` | ✅ | Monthly backups to keep |
| `type` | `"pg_dumpall"` \| `"wal_archive"` | ✅ | Backup type |

## Mandatory Databases

Soverstack requires these databases for core services:

| Database | Owner | Purpose |
|----------|-------|---------|
| `keycloak` | `keycloak` | IAM/SSO authentication |
| `headscale` | `headscale` | VPN coordination |
| `powerdns` | `powerdns` | DNS zone records |
| `openbao` | `openbao` | Secrets audit logs |
| `grafana` | `grafana` | Dashboard storage |

## Complete Example

### core-databases.yaml (Auto-generated)

```yaml
# DO NOT MODIFY - Auto-generated by Soverstack
# Documentation: https://docs.soverstack.io/mandatory-databases

databases:
  - type: postgresql
    version: "16"

    cluster:
      name: pg-core
      ha: true
      vm_ids: [250, 251, 252]    # From core-compute.yaml

    port: 5432
    ssl: required

    databases:
      - name: keycloak
        owner: keycloak

      - name: headscale
        owner: headscale

      - name: powerdns
        owner: powerdns

      - name: openbao
        owner: openbao

      - name: grafana
        owner: grafana

    credentials:
      type: vault
      path: secret/data/postgres/core

    backup:
      storage_backend: s3-backup
      schedule: "0 2 * * *"
      retention:
        daily: 7
        weekly: 4
        monthly: 12
      type: wal_archive
```

### databases.yaml (Your custom databases)

```yaml
# Your custom databases

databases:
  # Application database cluster
  - type: postgresql
    version: "16"

    cluster:
      name: pg-apps
      ha: true
      vm_ids: [250, 251, 252]    # Can share with core or use dedicated VMs

    port: 5432
    ssl: required

    databases:
      - name: myapp
        owner: myapp_user

      - name: analytics
        owner: analytics_user

      - name: reporting
        owner: reporting_user

    credentials:
      type: vault
      path: secret/data/postgres/apps

    backup:
      storage_backend: s3-backup
      schedule: "0 3 * * *"
      retention:
        daily: 7
        weekly: 4
        monthly: 6
      type: pg_dumpall
```

## HA Configuration

### Patroni + etcd

For `ha: true`, Soverstack deploys:
- **Patroni**: PostgreSQL HA manager
- **etcd**: Distributed consensus
- **HAProxy**: Load balancing

Minimum 3 VMs required for quorum.

### Failover

```
Primary (postgres-01) → Failure
        ↓
Patroni detects failure
        ↓
etcd leader election
        ↓
Replica promoted (postgres-02)
        ↓
HAProxy updates routing
```

## Validation Rules

### All Tiers

| Rule | Severity |
|------|----------|
| Unique cluster names | Critical |
| Unique database names within cluster | Error |
| Credentials required | Critical |
| Valid SSL mode | Error |

### Production/Enterprise

| Rule | Severity |
|------|----------|
| At least one cluster | Critical |
| `ssl: required` | Warning |
| HA enabled with 3+ VMs | Critical |
| Odd number of VMs for quorum | Warning |
| Mandatory databases present | Critical |
| Backup configured | Warning |

## Merge Behavior

When multiple files specified:

```yaml
database: "core-databases.yaml,databases.yaml"
```

| Property | Behavior |
|----------|----------|
| `databases` | Concatenated |

Duplicate cluster names will cause validation error.

## SSL Configuration

| Mode | Description | Tier |
|------|-------------|------|
| `required` | SSL mandatory | Production+ |
| `preferred` | SSL if available | Local |
| `disabled` | No SSL | Not recommended |

## Connecting to Databases

### From Services

Services reference databases by name:

```yaml
# In networking.yaml
vpn:
  database: headscale    # References core-databases.yaml

# In security.yaml
sso:
  database: keycloak     # References core-databases.yaml
```

### Connection String

```
postgresql://user:password@pg-core.internal:5432/myapp?sslmode=require
```
